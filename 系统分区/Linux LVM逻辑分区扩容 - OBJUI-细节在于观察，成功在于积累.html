Linux LVM逻辑分区扩容

17 2019年08月23日 Linux , 技术博客

                       

linux添加硬盘是很常见的，但硬盘不是插上去就可以直接使用，需要挂载。那么如何挂载呢，下面我们就使用LVM的方式挂载：

新加磁盘

[root@~] df -h
[root@~] fdisk -l
 fdisk /dev/sdb
     p （查看已经创建的分区）
     n (创建新分区)
     1 （分区号选择1）
     p （查看已经创建的分区）
     t （修改分区格式）
     8e （8e为Linux LVM格式）
     w 保存退出
[root@~] pvcreate /dev/sdb1  #（创建PV）
[root@~] pvdisplay #查看PV
[root@~] vgcreate vg0 /dev/sdb1  #创建VG
[root@~] vgdisplay  #查看VG
[root@~] lvcreate -L 8G -n lv1 vg0  #或 lvcreate -l 100%FREE -n lv1 vg0 
[root@~] lvdisplay
[root@~] mkfs.ext4 /dev/vg0/lv1 #使用ext4文件系统对LV进行格式化 
#将lv1挂载到/mnt/lv1目录下
[root@~] mkdir /mnt/lv1
[root@~] mount /dev/vg0/lv1 /mnt/lv1

直接添加到现有磁盘中

[root@~] df -h
[root@~] fdisk -l
[root@~] fdisk /dev/sdb
     p （查看已经创建的分区）
     n (创建新分区)
     1 （分区号选择1）
     p （查看已经创建的分区）
     t （修改分区格式）
     8e （8e为Linux LVM格式）
     w 保存退出
[root@~] pvcreate /dev/sdb1 #（创建PV）
[root@~] pvdisplay #查看PV
#vg0为VG Name
[root@~] vgextend vg0 /dev/sdb1
#将新的VG空间划入LV
[root@~] lvextend -l +100%FREE /dev/mapper/centos-root
#检查/root逻辑卷
[root@~] e2fsck -f /dev/mapper/centos-root
#重新定义分区大小
[root@~] resize2fs /dev/mapper/centos-root
# 刷新处理
[root@~] xfs_growfs /dev/mapper/centos-root

永久挂载

[root@~] vim /etc/fstab
/dev/sdb1(磁盘分区)  /data1（挂载目录） ext3（文件格式）defaults  0  0

其他操作

#安装LVM
[root@~] yum install lvm2  
#删除LV
[root@~] lvremove /dev/vg0/lv0
